# Regenerative Design Lens

Does this design regenerate or extract? This lens asks the foundational question before spatial, ecosystem, or compliance concerns. Adapted from Green Goods' regenerative framework for Coop's knowledge commons context.

---

## The Seven Principles — Coop Directives

### 1. Make the Mycelium Visible
Show value flows end-to-end. Every screen should help members trace the chain: source → agent extraction → draft → review → publish → archive. Name the flows. Make the invisible visible.

**Check**: Can the member see how their review decision connects to the larger system?

**Coop example**: The "Sourced from" section on draft cards traces knowledge from allowlisted sources through entity extraction to the specific recommendation. The "Track record" line shows how past decisions created precedents that inform current ones.

### 2. Design for Succession
New coops need simple templates and quick wins. Growing coops need source management and spending controls. Mature coops need cross-coop knowledge sharing and advanced governance. Match interface complexity to coop maturity.

**Check**: Is this feature appropriate for the coop's succession stage?

**Coop example**: The agent maturity model (Seedling → Sapling → Mature → Trusted) IS succession design — it matches capability to earned trust.

### 3. Enrich the Edges
The review queue, source registry, cross-coop boundary, and agent-human handoff are the richest design zones. Invest disproportionate attention there.

**Check**: If this feature sits at a stakeholder boundary, have you designed for bidirectional learning?

**Coop example**: The Chickens review flow is an edge — where agent recommendations meet human judgment. Rich provenance and precedent display makes this a learning boundary, not just an approval gate.

### 4. Failure is Succession
Rejection = clearing (show what could improve). Agent skip = restraint (show why it chose not to). Stale source = dormancy (not deletion — temporal invalidation). Frame negative states as opportunities with context.

**Check**: Does this error/empty/failure state guide toward renewal?

**Coop example**: Agent "skipped" decisions show reasoning in Decision History: "Low topic overlap with coop focus. Similar to rejected draft from March." This frames restraint as intelligence, not absence.

### 5. Be Growth-Agnostic
No streaks. No "days since last capture." No gamified entity counts. No FOMO about knowledge gaps. Celebrate task completion and departure. Measure knowledge quality, not quantity.

**Check**: Would removing this feature increase engagement? If yes, that's a feature, not a bug.

**Coop example**: Source stats show "47 entities" as context, not as a score to grow. No "add more sources!" urgency. The agent enriches what's there, not nags about what's missing.

### 6. Capability is the Deliverable
The goal is the coop's ability to make good decisions independently, not dependency on the agent. The agent builds shared understanding that persists. The Walkaway Test: if the agent stopped running, would the published knowledge and learned patterns remain useful?

**Check**: Does this build the coop's ability to function independently?

**Coop example**: Reasoning traces as precedent nodes mean the coop's decision history persists even if the agent's inference engine changes. The knowledge graph IS the institutional memory.

### 7. Regen, Not Degen
Natural light over financial terminal. Seasonal rhythms over countdown timers. Community narratives over individual dashboards. Garden journal aesthetic over trading floor aesthetic.

**Check**: Would this feel at home in a solarpunk world or a trading floor?

**Coop example**: Coop's warm cream/brown palette IS regen aesthetic. The brand direction explicitly says "warm, not corporate" and "observant, not surveillance-like."

## Regenerative Design Checklist

Run on new components alongside Spatial Readiness and Ecosystem Readiness:

```
Regenerative Design Check
│
├─ [ ] Value flow visible?
│      Member can trace how their action contributes to the larger system
│      (capture → agent → draft → review → publish → archive)
│
├─ [ ] Succession-appropriate?
│      Feature complexity matches coop maturity stage
│      (Seedling: simple · Sapling: configurable · Mature: advanced)
│
├─ [ ] Edge-enriched?
│      Stakeholder boundaries designed for learning, not just transaction
│      (Agent → human handoff shows reasoning, not just output)
│
├─ [ ] Failure as succession?
│      Error/empty/rejection states guide toward renewal with context
│      (Agent skips explained, source staleness shown, rejections teach)
│
├─ [ ] Growth-agnostic?
│      No engagement gamification, no urgency manufacturing
│      Metrics measure knowledge depth, not usage breadth
│
├─ [ ] Capability-building?
│      Increases community independence, not platform dependency
│      Passes the Walkaway Test (knowledge persists without agent)
│
├─ [ ] Regen aesthetic?
│      Warm Vellum material (cream, brown, green/orange accents)
│      No degen patterns (countdown timers, PnL dashboards, FOMO)
│
├─ [ ] Honorable Harvest?
│      Takes only the data it needs from allowlisted sources
│      Gives value back (enrichment, not just extraction)
│      Community controls what sources the agent can access
│
├─ [ ] Collective, not competitive?
│      Coop-level achievements, not individual leaderboards
│      Source sharing serves collective learning, not status
│
└─ [ ] Observant, not surveillance-like?
       Language uses "I noticed" not "I tracked"
       Passive observation feels curious and friendly, not aggressive
```

## The Degen/Regen Pattern Table

| Pattern | Degen | Regen (Coop) |
|---------|-------|------|
| Color palette | Dark mode, red/green financial | Warm cream, earth tones, green/orange |
| Primary metric | Entity count, API calls, tokens used | Knowledge quality, decision confidence |
| Time model | "X sources added this week!" | Natural cadence, refresh when needed |
| Social model | "Top contributor" leaderboards | Collective knowledge, shared sources |
| Ownership | "My agent, my sources" | "Our coop's knowledge commons" |
| Notifications | FOMO, re-engagement | Seasonal insights, completion celebration |
| Endgame goal | Retain the power user | Graduate the community |
| Value flow | Up and out (to platform) | Through and around (to all members) |
| Aesthetic | Financial terminal, productivity dashboard | Garden journal, warm paper, living knowledge |

## Motivation Design Filter

When adding any motivational element:

| Drive | Regenerative Use | Trap to Avoid |
|-------|-----------------|--------------|
| **Epic Meaning** | "This coop's knowledge helped 3 communities" | Abstract planetary messaging |
| **Development** | Source quality milestones, coop-level achievements | Individual badges, activity counts |
| **Creativity** | Custom source curation, skill creation | Platform-dependent creativity only |
| **Ownership** | "This is our shared knowledge" — communal stewardship | Individual portfolios, "my captures" |
| **Social** | Cross-coop source sharing, collective learning | Competitive comparison, social guilt |
| **Scarcity** | Replace with natural cadence (sources refresh when updated) | Countdown timers, limited offers |
| **Unpredictability** | Genuine discovery (agent finds unexpected connections) | Random rewards, variable reinforcement |
| **Loss** | Real signal (source went stale, knowledge gap identified) | Streak decay, progress loss, guilt |

**Rule**: Meaning, Creativity, and Social drives are self-sustaining. Scarcity, Unpredictability, and Loss create dependency. Always prefer the former.

# Hackathon Demo Voiceover Script

This is a literal narration script for the hackathon demo. It follows the current demo outline and
is written to be read almost verbatim.

For shot sequencing and operator actions, use
[Hackathon Demo Shot List](/reference/hackathon-demo-shot-list).

## Full Script

### Opening

We do not have an information problem. We have a fragmentation problem.

Important context already exists in messy tabs, notes, links, conversations, and half-finished
research — scattered across people and tools. What is usually missing is a way to turn that
scattered knowledge into clear opportunity, whether for yourself, a team, or an entire community.

That is what Coop is for.

### Landing And Framing

We start here on the landing page because this is a real product, not a concept deck.

Coop is browser-first, local-first, and passkey-first. The basic promise is simple: capture signal
where people already work, structure it locally, review it deliberately, and then share it into the
right collective context.

### Create The Third Coop

We already have two coops created here, and now we are creating a third live.

That matters because Coop is not only about collecting information. It is about routing knowledge
into the right group, the right purpose, and the right opportunity surface.

A coop is the shared coordination space. It gives a group a boundary for memory, review, archive,
and action.

### Round Up Messy Tabs

Now we move to a deliberately messy browser state.

This is how real work looks. Too many tabs, too many signals, too much context, and not enough
shared structure.

We run a roundup, and those tabs become chickens in a local review queue.

At this point nothing has been shared yet. The system is still local-first.

### Synthesis And Multi-Coop Routing

This is where Coop starts turning knowledge into opportunity.

The system is not just summarizing pages. It is looking at what matters, what kind of opportunity
or signal is present, and which coop it appears to belong to.

So now we can route different chickens into different coops based on purpose and fit.

That is the key shift: instead of one pile of browser clutter, we get structured findings moving
toward the right collective context.

The agent proposes. The human still confirms. Coop does not silently auto-publish AI output into
shared memory.

### Invite A Member

Next, we show how someone joins the coop.

Coop stays local-first and peer-to-peer in its posture. A trusted member can issue a scoped invite,
a new member joins through a passkey-first flow, and shared state syncs without turning the whole
system into a centralized cloud bucket.

That trust split matters. Not every member has the same authority, and not every action carries the
same risk.

### Connect To Green Goods

Now we move into the onchain coordination layer through Green Goods.

Green Goods is how Coop connects knowledge to governance, capital formation, capital allocation, and
impact reporting.

You can start a new garden with the coop, or connect the coop to an existing garden structure by
adding it into the right ownership or operator path.

This is important because the coop is not just a memory tool. It is a coordination tool that can
connect what the group knows to what the group can actually do.

### Governance And Bounded Session-Key Action

Here is the governance beat.

The coop agent can synthesize the group's priorities and suggest that it would make sense to create
a pool aligned with those priorities.

Then, through a bounded session-key path, the system can execute that action in a scoped way.

That is the important distinction. This is not open-ended autonomy. It is bounded initiative.

The system can help move from shared understanding into real capital-allocation and governance
infrastructure, while still keeping policy, role, and execution scope legible.

### Filecoin Close

Finally, we close on the archive path.

Coop data stays local-first and syncs peer-to-peer where appropriate, but it can also be backed up
through a decentralized Filecoin archive path.

That means the coop's knowledge does not have to disappear into ephemeral tools, screenshots, and
private memory. It can become durable, portable, and provable.

If encrypted archive is enabled in the path we are showing, then durability also does not require
giving up control over the contents.

### Close

So the full loop is:

create the coop, round up messy knowledge, route that knowledge into the right collective contexts,
invite people into the shared trust boundary, connect to Green Goods for onchain coordination and
capital formation, and then back the coop's memory up through Filecoin.

That is Coop.

It turns messy knowledge into opportunity, coordinated governance, and durable public memory.

### Future Coda

What comes next is deeper knowledge exploration, stronger Coop OS patterns, richer mobile and
call-based participation, and more ways to turn what you know into coordinated opportunity.

## Shorter Alt Close

If you want a sharper final line:

Coop helps you move from scattered context to shared opportunity, from shared opportunity to
governance, and from governance to durable memory.

## Delivery Notes

- keep the pacing calm and declarative
- do not over-explain implementation details while the UI is moving
- emphasize "local-first", "peer-to-peer", "bounded", and "opportunity" at the moments where they
  matter most
- if a rail is in rehearsal or mock mode, say so directly and move on
